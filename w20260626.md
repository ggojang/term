### 2. TC_CONCEPT_STATS 전체 재계산 (필수)

**이유:** BFS 기반 자손 수 계산 알고리즘으로 교체됨 (이전 Kahn's 위상정렬 방식은 DAG에서 중복 집계 발생). 기존 데이터 삭제 후 재계산 필요.

#### 2-1. 기존 TC_CONCEPT_STATS 데이터 삭제

```sql
-- psql 접속
psql -U postgres -d term

-- 전체 삭제
DELETE FROM term.tc_concept_stats;
COMMIT;

-- 확인
SELECT COUNT(*) FROM term.tc_concept_stats;  -- 0이어야 함
```

#### 2-2. TC_CONCEPT_STATS 재계산 배치 실행

```bash
cd ~/services/term

# 전체 96개 릴리즈 재계산 (백그라운드 실행)
bash build_tc_batch.sh >> /tmp/tc_batch.log 2>&1 &
echo "PID: $!"

# 진행 상황 확인
tail -f /tmp/tc_batch.log

# 완료 확인 (96이면 완료)
grep -c "TC 완료" /tmp/tc_batch.log
```

**예상 소요 시간:** 릴리즈당 40~80초 × 96개 = 약 1.5~2시간

#### 2-3. 완료 후 검증

```sql
-- 96개 릴리즈 모두 있는지 확인
SELECT COUNT(DISTINCT effective_time) FROM term.tc_concept_stats;
-- 결과: 96

-- 루트 개념 자손 수 확인 (최신 릴리즈)
SELECT effective_time, descendant_count
FROM term.tc_concept_stats
WHERE concept_id = '138875005'
ORDER BY effective_time DESC
LIMIT 5;
-- 최신(20260601) 약 380,756이면 정상
```

### 4. 주의 사항

- TC_CONCEPT_STATS 배치 실행 중에도 서비스 운영 가능 (처리된 릴리즈부터 정상 표시됨)
